Домашнее задание к занятию "5.2. Системы управления виртуализацией"

Задача 1.
Выберете подходящую систему управления виртуализацией для предложенного сценария.
Детально опишите ваш выбор.
Сценарии:
    - 100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых
    требований. Преимущественно Windows based инфраструктура, требуется
    реализация программных балансировщиков нагрузки, репликации данных и
    автоматизированного механизма создания резервных копий;

vSphere. Мощный продукт со всеми требуемыми возможностями.

    - Требуется наиболее производительное бесплатное opensource решение для
    виртуализации небольшой (20 серверов) инфраструктуры Linux и Windows
    виртуальных машин;

KVM или XEN. Открытые, стабильные и производительные системы.

    - Необходимо бесплатное, максимально совместимое и производительное решение
    для виртуализации Windows инфраструктуры;

Hyper-V как Windows ориентированная система.

    - Необходимо рабочее окружение для тестирование программного продукта на
    нескольких дистрибутивах Linux.

VirtualBox или VMware Workstation. Для нескольких дистрибутивов нет нужды
разворачивать тяжеловесную систему и тем более делать выбор в пользу платных.

Задача 2.
Опишите сценарий миграции с VMware vSphere на Hyper-V для Linux и Windows
виртуальных машин. Детально опишите необходимые шаги для использования всех
преимуществ Hyper-V для Windows.

Первый способ - использовать средства конвертации, встроенные в Virtual Machine
Manager (VMM).
Второй способ - использовать отдельную утилиту Microsoft Virtual Machine
Converter. Сторонние производители ПО (превосходят средства, предлагаемые
компанией Microsoft, как в скорости конвертации, так и в удобстве использования),
например StarWind V2V Converter и 5nine V2V Easy Converter.

Диски форматов VMDK, VHD, VHDX на деле мало чем отличаются друг от друга. В их
основе RAW c различными добавками. HEX-редактором или функциями ОС для работы с
файловой системой можно превратить VMDK или VHDХ в диск целевого гипервизора.

Преимущества Hyper-V для Windows:
Привычный интерфейс, широкая аппаратная совместимость (составной компонент
системы Windows).
Низкие затраты на оборудование и программное обеспечение. Благодаря технологии
виртуализации серверов Microsoft Hyper-V, на каждом физическом сервере может
быть запущено несколько виртуальных серверов, используя одну лицензию на Windows
Server Enterprise или Datacenter.
Чтобы использования Hyper-V необходимо соблюсти следующие требования:
    - Windows 10 Enterprise, Pro, или Education.
    - 64-битный процессор с Second Level Address Translation (SLAT).
    - Поддержка CPU для VM Monitor Mode Extension (VT-c на Intel CPUs).
    - Как минимум 4 GB памяти.

Задача 3

Опишите возможные проблемы и недостатки гетерогенной среды виртуализации
(использования нескольких систем управления виртуализацией одновременно) и что
необходимо сделать для минимизации этих рисков и проблем. Если бы у вас был бы
выбор, то создавали ли вы бы гетерогенную среду или нет? Мотивируйте ваш ответ
примерами.

Гетерогенные виртуальные среды могут быть неэффективны и сложны в управлении.
Проблемы возникают и на этапе интеграции решений по виртуализации от различных
производителей. Отсутствие каких-либо стандартов приводит к тому, что эти решения
либо вообще не интегрируются (точнее, интегрируются, но ни одним из производителей
не поддерживаются), либо требуют для интеграции дополнительный слой нестандартных
решений, что приводит к гетерогенности всей системы в целом, снижению надежности
или к необходимости построения всей инфраструктуры на решениях от одного
производителя для устранения проблем несовместимости.
В гетерогенных средах, когда приложения размещены в разных средах виртуализации,
каждой такой средой приходится управлять собственным инструментом. Что довольно
сложно, трудоемко и может значительно усложнить задачи управления и мониторинга.

Что необходимо сделать для минимизации этих рисков и проблем?
Гетерогенность труднее в освоении. Но предоставляет больше возможностей для
изменения архитектуры в будущем.
Важно помнить о наличии у сотрудников определенных навыков и в случае
необходимости производить своевременное обучение.

Если бы у вас был бы выбор, то создавали ли вы бы гетерогенную среду или нет?
Скорее нет. По возможности использовал бы open source решения, Linux, KVM и XEN.
Гомогенная инфраструктура обычно проще с точки зрения инсталляции, настройки,
использования и управления.
